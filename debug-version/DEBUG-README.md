# 🔧 调试版本说明

## 启动调试版本
```bash
./debug-start.sh
```

## 调试功能
- **调试标识**: 页面右上角显示"🔧 调试模式"标识
- **控制台日志**: 启动时输出详细调试信息
- **测试数据**: 可快速加载/清空测试数据

## 可用调试命令
在浏览器控制台中使用：

### 数据管理
```javascript
// 加载测试数据（包含多个月的记录）
loadTestData()

// 清空所有数据
clearTestData()

// 导出当前数据
exportData()

// 查看当前数据结构
console.log(window.tracker.data)
```

### 调试技巧
1. **快速测试**: 打开页面后立即运行 `loadTestData()` 查看完整功能
2. **数据检查**: 使用 `console.log(tracker.data)` 查看数据结构
3. **重置测试**: 使用 `clearTestData()` 清空后重新测试
4. **导出备份**: 测试前导出数据，测试后可恢复

## 与正式版本的区别
- ✅ 显示调试标识和日志
- ✅ 控制台输出详细信息
- ✅ 专用调试启动脚本
- ✅ 独立的数据存储（不影响正式版本）

## 端口配置
- 默认端口: 3001
- 备用端口: 3002（当3001被占用时）
- 正式版本使用: 3000

## 开发建议
1. 在调试版本中测试新功能
2. 确认无误后复制到正式版本
3. 保持两个版本的核心逻辑同步
